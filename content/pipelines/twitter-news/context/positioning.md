## What is Grass

Grass is a machine built for AI coding agents. One place where every agent lives — always on, always reachable, always ready to run. It gives every developer an always-on cloud VM with Claude Code, Codex, and Open Code pre-loaded, accessible from any surface.

## Positioning statement

For developers running AI coding agents like Claude Code and Codex in production workflows, who need to orchestrate multiple agents across the gaps in their day, Grass is a machine built for AI coding agents that runs every agent on an always-on cloud VM you can dispatch to from your laptop, your phone, or an automation. Unlike juggling separate agent tools on a sleep-prone laptop, Grass gives you one surface for every agent, keeps your work alive when you're not there, and stays neutral across models so you're never locked in.

## Competitive framing

**We compete against:** The fragmented reality of running multiple agents today — tmux + Tailscale + SSH into a personal server; juggling Claude Code in one window and Codex in another; waiting to get back to your desk to check on a long-running task; generic cloud VMs set up from scratch every time.

**We do NOT compete against:** Claude Code's mobile app, Codex's desktop UI, Cursor. Choosing this frame is what lets Grass be a category of one rather than "a mobile app for Claude Code."

## Unique attributes

- **Agent-agnostic by design** — Claude Code, Codex, Open Code, and the next ten agents all live here as first-class citizens
- **Always-on cloud VM per user** — powered by Daytona; agents don't die when your laptop sleeps
- **Pre-loaded and zero-setup** — your stack is ready in seconds, not configured over a weekend
- **Multi-surface access** — laptop (MCP dispatch), phone (mobile app), automation (scheduled/triggered)
- **BYOK authentication** — developers own their keys, no OAuth token exploitation
- **Skeuomorphic, machine-feel design** — you operate it like a Tesla app, not stare at it like a dashboard

## Target customer

Developers who already use AI coding agents seriously — multi-hour autonomous tasks, parallel repos, real production work — and who are starting to feel the friction of running them across fragmented tools and a single sleep-prone laptop. Early adopter profile: has used Claude Code or Codex for 2+ months, has tried at least two agents, has complained about the laptop-tether.

## Jobs to be done

- **Primary:** "When I'm running AI coding agents seriously and want to keep work flowing across the gaps in my day, I want one machine where every agent lives and that I can reach from anywhere, so I can keep shipping without losing momentum."
- Run parallel agents on different repos from one surface
- Try new coding agents without rebuilding your workflow
- Fire off long-running agent tasks and forget them
- Dispatch work immediately when an idea strikes away from your desk
- Get your agent stack ready instantly on a new project

## Canonical phrases

Use these verbatim and consistently across all content:

- "A machine built for AI coding agents"
- "Always-on cloud VM"
- "Agent-agnostic" / "agent-neutral"
- "One surface. Every agent. Always on."
- "Claude Code, Codex, Open Code" (specific agent names build entity associations)
- "BYOK"
- "Dispatch" (pair with concrete verbs like "fire off" or "send tasks")

## What NOT to say

- Don't lead with "mobile" or "phone" in headlines — that primes the wrong competitive frame
- Don't say "Claude Code on your phone" — this is explicitly rejected positioning
- Don't paraphrase the canonical phrases; repetition builds entity associations in model weights
- Mobile is one of three surfaces, not the identity
